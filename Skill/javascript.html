Pertanyaan seputar JavaScript

Banyak developer yang memilih menggunakan bahasa Javascript ini karena dengan bahasa ini nantinya dapat membuat website yang dinamis. Nantinya, website yang dibuat dapat memuat sebuah konten tanpa mereload halaman.

Selain itu, JavaScript memiliki dukungan dari komunitas. Perlu diketahui bahwa dukungan komunitas menjadi hal yang sangat penting untuk memilih bahasa pemrograman. Ketika kamu membuat atau mengembangkan aplikasi, terkadang akan menemukan bug dan dapat mencari solusi dari bug tersebut di komunitas.

  • Universal (hampir mendukung semua browser)
  • Menjadikan frontend lebih powerfull (menjadi dinamis)
  • Bahasa dari banyak aplikasi (contohnya SSR dan Aplikasi Mobile)
  • Mudah dipelajari
  • Membuat sebuah website menjadi lebih hidup

    • Ya, Bisa. Anda dapat membuat user interface menggunakan JavaScript dan kemudian menggunakannya untuk mengontrol back-end situs web Anda. Anda dapat menggunakan JavaScript sebagai back-end dengan menggunakan Node.js dan Express.js.

    Node.js merupakan platform untuk membuat aplikasi jaringan yang scalable. Node.js dapat digunakan untuk menjalankan server web, membuat aplikasi FastCGI, dan membangun aplikasi real-time.
    Express.js adalah framework yang kuat untuk membangun aplikasi web. Express.js merupakan framework yang cepat, murah, dan mudah digunakan. Bersama-sama, Anda dapat menggunakan teknologi ini untuk membuat back-end untuk situs web kita menggunakan JavaScript.

    Pengenalan JavaScript

    JavaScript, sering disingkat JS, adalah bahasa pemrograman yang merupakan salah satu teknologi inti dari World Wide Web, di samping HTML dan CSS. Ini memungkinkan Anda menambahkan interaktivitas ke halaman, misalnya Anda mungkin telah melihat sliders, alerts, interaksi klik, popup, dll di situs web yang berbeda — semuanya dibuat menggunakan JavaScript. Selain digunakan di browser, ini juga digunakan di lingkungan non-browser lainnya seperti Node.js untuk menulis kode sisi server dalam JavaScript, Electron untuk menulis aplikasi desktop, React Native untuk aplikasi seluler, dan sebagainya.

    • Sejarah JavaScript

    JavaScript awalnya dibuat oleh Brendan Eich dari NetScape dan pertama kali diumumkan dalam siaran pers oleh Netscape pada tahun 1995. JavaScript memiliki sejarah penamaan yang aneh; awalnya diberi nama Mocha oleh pembuatnya, yang kemudian diganti namanya menjadi LiveScript. Pada tahun 1996, sekitar setahun kemudian setelah rilis, NetScape memutuskan untuk mengganti namanya menjadi JavaScript dengan harapan memanfaatkan komunitas Java (walaupun JavaScript tidak memiliki hubungan apa pun dengan Java) dan merilis Netscape 2.0 dengan dukungan resmi JavaScript.

    • Deklarasi Variabel

    Untuk menggunakan variabel dalam JavaScript, pertama-tama kita harus membuatnya yaitu mendeklarasikan variabel. Untuk mendeklarasikan variabel, kita menggunakan salah satu kata kunci var, let, atau const.

  • Var

    Statement var mendeklarasikan variabel dengan cakupan fungsi atau cakupan global, secara opsional menginisialisasinya ke sebuah nilai. Tetapi sekarang banyak developer menghindari penggunaan variabel var.

  • Const

    Const memiliki cakupan blok, seperti halnya variabel yang dideklarasikan menggunakan let. Nilai const tidak dapat diubah melalui penugasan ulang (yaitu dengan menggunakan operator penugasan), dan tidak dapat dideklarasikan ulang (yaitu melalui deklarasi variabel). Namun, jika const adalah objek atau larik, properti atau itemnya dapat diperbarui atau dihapus.

    • Variabel Scopes

    Dalam JavaScript, scopes mengacu pada visibilitas variabel atau bagaimana itu dapat digunakan setelah dideklarasikan. Scopes variabel tergantung pada kata kunci yang digunakan untuk mendeklarasikannya.

    Tiga jenis scope adalah Scope Global, Scope Fungsi, dan Scope Blok. Sebelum ES6 (2015), JavaScript hanya memiliki Scope Global dan Scope Fungsi dengan var. ES6 diperkenalkan let dan const yang memungkinkan Block Scope dalam JavaScript.

    Scope Global: Variabel yang dideklarasikan di luar fungsi apa pun atau kurung kurawal '{}' memiliki Cakupan Global, dan dapat diakses dari mana saja dalam kode Javascript yang sama. Var, let dan const semuanya menyediakan Cakupan ini.

    Scope Fungsi: Variabel yang dideklarasikan dalam suatu fungsi hanya dapat digunakan dalam fungsi yang sama. Di luar fungsi itu, mereka tidak terdefinisi. var, let dan const semuanya menyediakan Cakupan ini.

    Scope Blok: Blok adalah bagian apa pun dari kode JavaScript yang dibatasi oleh '{}'. Variabel yang dideklarasikan di dalam blok tidak dapat diakses di luar blok itu. Lingkup ini hanya disediakan oleh kata kunci let dan const. Jika Anda mendeklarasikan variabel di dalam blok menggunakan var, itu TIDAK akan memiliki Cakupan Blok.

    • Operator Pembanding Nilai

    Dalam javascript, operator == melakukan konversi tipe operan sebelum perbandingan, sedangkan operator === membandingkan nilai dan tipe data operan. Metode Object.is() menentukan apakah dua nilai adalah nilai yang sama: Object.is(value1, value2).

    Object.is() tidak setara dengan operator ==. Operator == menerapkan berbagai paksaan ke kedua sisi (jika keduanya bukan tipe yang sama) sebelum menguji kesetaraan (menghasilkan perilaku seperti "" == false being true), tetapi Object.is() tidak memaksakan salah satu nilai.

    Object.is() juga tidak setara dengan operator ===. Satu-satunya perbedaan antara Object.is() dan === adalah dalam perlakuan mereka terhadap nol dan nilai NaN. Operator === (dan operator ==) memperlakukan nilai angka -0 dan +0 sebagai sama tetapi memperlakukan NaN tidak sama satu sama lain.

    • Statemen Kondisi

    Saat Anda menulis kode, Anda sering kali ingin melakukan tindakan yang berbeda untuk keputusan yang berbeda. Anda dapat menggunakan pernyataan bersyarat dalam kode Anda untuk melakukan ini. Dalam JavaScript, kami memiliki tiga pernyataan bersyarat: if, if...else, dan switch.

    if...else

    Statemen if mengeksekusi pernyataan jika kondisi yang ditentukan adalah truthy. Jika kondisinya falsy, pernyataan lain dalam klausa else opsional akan dieksekusi.

    if (condition) {
    statement1;
    } else {
    statement2;
    }

    Switch

    Statemen switch mengevaluasi ekspresi, mencocokkan nilai ekspresi dengan serangkaian klausa case, dan mengeksekusi pernyataan setelah klausa case pertama dengan nilai yang cocok, hingga statemen break ditemukan. Klausa default statemen switchakan dilompati jika case tidak cocok dengan nilai ekspresi.

    Example

    switch (expression) {
    case value1:
    //Statements executed when the result of expression matches value1
    break;
    case value2:
    //Statements executed when the result of expression matches value2
    break;
    ...
    case valueN:
    //Statements executed when the result of expression matches valueN
    break;
    default:
    //Statements executed when none of the values match the value of the expression
    break;
    }

    • Exception Handling

    Dalam JavaScript, semua pengecualian hanyalah objek. Meskipun sebagian besar pengecualian adalah implementasi dari kelas Error global, objek lama apa pun dapat dilempar. Dengan mengingat hal ini, ada dua cara untuk melontarkan pengecualian: langsung melalui objek Error, dan melalui objek kustom. (exception dari Rollbar)

    Statemen Throw

    Statement throw melempar pengecualian user-defined. Eksekusi fungsi saat ini akan berhenti (pernyataan setelah throw tidak akan dieksekusi), dan kontrol akan diteruskan ke blok catch pertama di call stack. Jika tidak ada blok catch di antara fungsi pemanggil, program akan berhenti. (kutipan dari MDN)

    Try, Catch, Finally

    Ini adalah cara menangani kesalahan dalam kode JavaScript Anda. Di dalam blok kode try kita memiliki kode untuk dijalankan, di dalam blok catch kita menangani kesalahan, dan di dalam blok finally kita memiliki kode yang berjalan setelah eksekusi blok kode sebelumnya, apa pun hasilnya.

    Utilizing Error Objects

    Ketika kesalahan runtime terjadi, objek error baru dibuat dan dilempar. Dengan objek error ini, kita dapat menentukan jenis Error dan menanganinya sesuai dengan jenisnya.

    Jenis error:

    Selain konstruktor error, Javascript juga memiliki konstruktor Error inti lainnya.

  • AggregateError
  • EvalError
  • InternalError
  • RangeError
  • ReferenceError
  • SyntaxError

    • Example

    try {
    willGiveErrorSometime();
    } catch (error) {
    if (error instanceof RangeError) {
    rangeErrorHandler(error);
    } else if (error instanceof ReferenceError) {
    referenceErrorHandle(error);
    } else {
    errorHandler(error);
    }
    }

    • Expression & Operator

    Pada tingkat tinggi, ekspresi adalah unit kode yang valid yang menghasilkan nilai. Ada dua jenis ekspresi: yang memiliki efek samping (seperti menetapkan nilai) dan yang murni mengevaluasi. Ekspresi x = 7 adalah contoh dari tipe pertama. Ekspresi ini menggunakan operator = untuk menetapkan nilai tujuh ke variabel x. Ekspresi itu sendiri bernilai 7. Ekspresi 3 + 4 adalah contoh dari tipe kedua. Ekspresi ini menggunakan operator + untuk menjumlahkan 3 dan 4 bersama-sama dan menghasilkan nilai, 7. Namun, jika pada akhirnya bukan bagian dari konstruksi yang lebih besar (misalnya, deklarasi variabel seperti const z = 3 + 4), hasilnya akan segera dibuang - ini biasanya kesalahan programmer karena evaluasi tidak menghasilkan efek apa pun. Seperti yang juga diilustrasikan oleh contoh di atas, semua ekspresi kompleks digabungkan oleh operator, seperti = dan +.

    • Function Parameters

    Parameter adalah nama yang diberikan kepada variabel yang dideklarasikan di dalam definisi suatu fungsi. Ada dua jenis sintaks khusus: default dan rest parameter.

    Default Params

    Parameter fungsi default memungkinkan parameter bernama diinisialisasi dengan nilai default jika tidak ada nilai atau undefined.

    Rest Params

    Sintaks parameter lainnya memungkinkan suatu fungsi untuk menerima argumen dalam jumlah tak terbatas sebagai larik, menyediakan cara untuk merepresentasikan fungsi variadik dalam JavaScript.

    • Built-in Function

  • Methode JavaScript adalah properti yang berisi definisi fungsi. Dengan kata lain, ketika data yang disimpan pada suatu objek adalah fungsi, kita menyebutnya sebagai metode.

  • Untuk membedakan antara properti dan metode, kita dapat memikirkannya seperti ini: Properti adalah apa yang dimiliki objek, sedangkan metode adalah apa yang dilakukan objek.
